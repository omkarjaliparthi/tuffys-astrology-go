# Security Policy

## Reporting a vulnerability

If you discover a security issue in the Kriya Go SDK or in the API
surfaces it talks to, please report it privately to:

**admin@insightsbyomkar.com**

Please include:

- A description of the issue and where it lives
- Steps to reproduce, ideally with a minimal repro
- Any relevant logs, requests, or proof-of-concept
- Your contact info so we can follow up

We aim to acknowledge new reports within **72 hours** and to share
an initial assessment within **7 days**.

## Scope

This policy covers:

- The code in this repository (the Kriya Go SDK)
- The Kriya API surfaces at `kriya.insightsbyomkar.com` that this SDK
  talks to

For other Insights by Omkar properties, see the security policy on
each repo or the studio site at
[studio.insightsbyomkar.com](https://studio.insightsbyomkar.com).

## Disclosure

We follow coordinated disclosure. Once a fix is in place we will
work with you on timing for public disclosure and will credit
reporters who want to be credited.
